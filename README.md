# sip-order - 飲料訂購程式

以 Google Apps Script (GAS) 驅動的現代化、響應式飲料訂購系統。前端採用單一 HTML (RWD CSS + JavaScript)，後端直接連結 Google 試算表（Google Sheets）進行訂單資料儲存。

## 專案特色
- **無伺服器架構**：完全基於 Google 雲端硬碟生態系，免租用伺服器與資料庫。
- **響應式設計 (RWD)**：支援手機、平板與電腦瀏覽，隨時隨地輕鬆點餐。
- **精美 UI**：阿芝精心設計的現代化飲料點單介面，包含滑順的微動畫與視覺反饋。
- **簡單部署**：將程式碼複製貼上至 Google Apps Script 即可完成部署。

## 訂購表單欄位
本程式前端提供以下點單資訊輸入：
1. **訂購者**：輸入訂購人姓名。
2. **飲料品項**：選擇或輸入喜好的飲料（例如：珍珠奶茶、四季春茶）。
3. **甜度**：提供「無糖、一分糖、三分糖、半糖、七分糖、全糖」六種選擇。
4. **冰量**：提供「去冰、少冰、多冰、全冰」四種選擇。
5. **數量**：設定訂購杯數。

## 部署與使用方式 (第一方案：手動複製部署)

由於本專案採用無 Clasp 本地同步之手動複製部署方式，請按照以下步驟完成設定：

### 步驟 1：建立 Google 試算表
1. 前往 [Google 雲端硬碟](https://drive.google.com/)。
2. 新增一個「Google 試算表」，並為其命名（例如：`sip-order 訂單管理`）。
3. 在試算表的第一列（A1 到 G1）建立以下欄位作為標題（非強制，但有助於閱讀）：
   - `時間戳記` | `訂購者` | `飲料品項` | `甜度` | `冰量` | `數量` | `狀態`

### 步驟 2：開啟 Apps Script 編輯器
1. 在剛建立的 Google 試算表中，點選上方選單的 **「延伸模組」 (Extensions)** -> **「Apps Script」**。
2. 這會開啟 Google Apps Script 程式碼編輯器。

### 步驟 3：複製程式碼
1. 將本專案根目錄下 `Code.gs` 的內容全部複製，貼上並覆蓋 Apps Script 中的 `代碼.gs` (或 `Code.gs`)。
2. 在 Apps Script 編輯器左側，點選 **「+」** 新增檔案，選擇 **「HTML」**，並命名為 `Index`（請確保大小寫一致，不要加副檔名 `.html`，Apps Script 會自動將其命名為 `Index.html`）。
3. 將本專案根目錄下 `Index.html` 的內容全部複製，貼上並覆蓋剛建立的 `Index.html` 編輯區。
4. 按下上方 **「儲存項目」 (Save project)** 圖示儲存所有變更。

### 步驟 4：部署為網頁應用程式 (Web App)
1. 點選編輯器右上角的 **「部署」 (Deploy)** -> **「新增部署」 (New deployment)**。
2. 點選左上角齒輪圖示，選擇 **「網頁應用程式」 (Web App)**。
3. 填寫說明（例如：`sip-order v1.0`）。
4. **執行身分 (Execute as)**：選擇 **「我」 (Me)**。
5. **誰有權限存取 (Who has access)**：選擇 **「任何人」 (Anyone)**。
   *註：若要讓所有訂購者（即使沒登入 Google 帳號）都能點單，必須選擇「任何人」。*
6. 點選 **「部署」 (Deploy)**。
7. 首次部署時，Google 會要求授權（因程式需要寫入試算表），請點選「授予存取權」，選擇您的 Google 帳戶，點選「進階」並選擇「前往『未命名專案』（安全偏離）」，然後點選「允許」。
8. 部署成功後，複製畫面上的 **「網頁應用程式 URL」**。這就是您的飲料訂購網址！
